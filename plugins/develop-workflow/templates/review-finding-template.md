# 리뷰 지적사항 템플릿

리뷰 결과는 아래 구조를 사용한다.

## 지적사항

각 지적사항은 아래 형식을 사용한다.

```text
[심각도] 제목
근거: file:line과 관찰한 정확한 동작.
영향: 구체적인 실패, 악용 가능성, 병목, 유지보수 리스크.
수정: 코드 수준의 구체적인 변경.
테스트: 수정이 증명되어야 하는 시나리오 또는 명령.
```

심각도 값:

- Critical: 치명
- High: 높음
- Medium: 중간
- Low: 낮음

문제가 없을 때:

```text
차단 수준의 지적사항 없음.
잔여 리스크: 런타임 설정, 운영 권한, 부하 동작처럼 검증하지 못한 영역.
권장 검증: 실행할 명령 또는 수동 확인 항목.
```

규칙:

- 지적사항을 먼저 쓰고 심각도 순서로 정렬한다.
- 정확성, 보안, 성능, 유지보수성에 영향을 주지 않는 단순 스타일 선호는 지적사항으로 올리지 않는다.
- 가능하면 파일과 라인 참조를 포함한다.
- 다른 엔지니어가 바로 구현할 수 있을 만큼 구체적으로 권장사항을 작성한다.
