# 요구사항 명확화

## 설명

사용자 요청에 백엔드 의도는 있지만 구현 결정을 안전하게 가정할 수 없을 때 사용한다. 범위, 대상 모듈, 성공 기준, API 동작, 데이터 소유, 보안 경계, 마이그레이션 위험, 검증 기준을 명확히 한다.

## 실행 절차

1. target이 언급되면 관련 코드나 repo 구조를 먼저 확인한다.
2. 현재 이해를 짧은 문단으로 정리한다.
3. 구현 결정을 바꾸는 질문만 남긴다.
4. scope, caller, target module, data ownership, authorization, edge case, performance constraint, rollout, validation을 확인한다.
5. 안전하게 진행하는 데 필요한 최소 질문만 한다.

## 검증

- 질문이 실제 구현 선택을 바꾸는지 확인한다.
- 질문 수가 과도하면 scope, 보안 경계, 성공 기준 중심으로 줄인다.

## 주의사항

- 실수 방지 가드레일: 모호한 요청은 구현하지 말고 성공 기준, 보안 경계, 검증 방법을 먼저 확정한다.
- 답을 가정하고 구현하지 않는다.
- 질문은 추상적이면 안 되고, 구현 선택지를 좁혀야 한다.

## 출력

다음 형식으로 반환한다.

```text
현재 이해:

확인 질문:
1.
2.

확인한 관련 맥락:
- file/path or module:
```
