# 설계 결정 템플릿

백엔드 아키텍처나 구현 방향을 계획할 때 아래 구조를 사용한다.

```text
목표:
성공 기준:
제약 조건:

제안 설계:
- 경계:
- 데이터 흐름:
- 트랜잭션 모델:
- 인가 모델:
- 오류 모델:
- 성능 고려사항:
- 관측 가능성:

검토한 대안:
- 대안:
- 트레이드오프:

검증:
- 단위 테스트:
- 통합 테스트:
- 보안 점검:
- 성능 점검:
```

규칙:

- 의존성 방향을 명시한다.
- 입력 검증, 인가, 트랜잭션 제어, 영속성, 외부 호출을 어느 계층이 소유하는지 명시한다.
- 성공 기준을 만족하는 가장 작은 설계를 우선한다.
